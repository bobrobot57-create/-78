# Закрепление админов

## Проблема

Партнёр получал «Нет доступа» — его ID не был в списке админов.

## Решение

В переменной **ADMIN_USER_IDS** перечисляются все ID с полным доступом через запятую.

## Текущие админы

| Кто | @username | ID |
|-----|-----------|-----|
| Создатель | @Old_Bobr57 | 1203677805 |
| Партнёр | @Drykey | 1458925724 |

## Что сделать

### В Railway (Variables)

Добавь или измени переменную:

```
ADMIN_USER_IDS=1203677805,1458925724
```

### В .env (локально)

```
ADMIN_USER_IDS=1203677805,1458925724
```

### Перезапуск

После изменения переменных перезапусти деплой на Railway (Redeploy).

---

Оба ID получат полный доступ к админ-боту. Первый в списке (1203677805) считается владельцем для отображения в интерфейсе.
